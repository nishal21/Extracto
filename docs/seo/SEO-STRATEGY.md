# Extracto SEO Strategy

## 1. Discovery & Objectives
- **Business Type:** Open-source / SaaS Developer Tool
- **Target Audience:** Developers, Data Scientists, Growth Hackers, Researchers
- **Core Value Proposition:** Plain English AI web scraping that bypasses JS rendering issues and natively handles legacy routing constraints without the need for manual CSS selectors.
- **KPIs:**
  - Organic traffic to `/docs`
  - GitHub Stars generated from organic search
  - `pip install extracto-scraper` growth

## 2. Competitive Intelligence
- **Tier 1 Competitors:** Apify, Crawlee, Scrapy
- **Tier 2 Competitors:** ScrapeGraphAI, Beautiful Soup, Selenium
- **Keyword Gaps & Opportunities:**
  - Most competitors rank for "how to scrape [website] with Python". Extracto can dominate long-tail "AI web scraping without selectors" or "scrape JS rendered sites with LLM".
  - Competitors lack modern "Generative Engine Optimization" features.

## 3. SEO Keyword Strategy
### Primary Keywords (High Volume, High Competitiveness)
- AI Web Scraper
- Python LLM Scraper
- No-code Python Scraper
- Playwright Web Scraper

### Secondary/Long-Tail Keywords (High Intent)
- How to scrape SPAs with Python
- Scrape onclick javascript links
- Extracto vs ScrapeGraphAI
- Extracto vs Apify
- Convert any website to JSON Python

## 4. E-E-A-T Strategy
- **Experience & Expertise:** Publish deep-dive technical blogs showcasing how Extracto handles edge cases (like `onclick="window.open()"` routing) that break traditional scrapers.
- **Authoritativeness:** Integrate badges for PyPI downloads, GitHub stars, and prominent "Built with ❤️" links.
- **Trustworthiness:** Transparent open-source code, clear MIT license, exhaustive SDK documentation.

## 5. Content Pillars
1. **Tool Comparisons** (`/compare/extracto-vs-[competitor].html`)
2. **Use Cases** (`/use-cases/[industry]-scraping.html`)
3. **Tutorials/Blog** (`/blog/how-to-[action].html`)
